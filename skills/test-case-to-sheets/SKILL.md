# Markdown 测试用例转飞书表格

将 `test-case-generator` 输出的 Markdown 测试用例文件转换为飞书电子表格，自动补全缺失字段并处理格式差异。

## 转换流程

### 1. 解析 Markdown 文件

读取 Markdown 文件，提取以下信息：

- **frontmatter**：`program`（项目名）、`module`（功能模块）、`platform`（平台分类）
- **章节结构**：识别 `##` 和 `###` 层级关系，判断是单层还是多层章节
- **用例表格**：逐行提取表格中的「模块」「标题」「前置条件」「测试步骤」「预期结果」「优先级」

### 2. 字段映射与推导

| 飞书表格字段 | 来源 | 推导规则 |
|----------|------|----------|
| 用例名称 | Markdown 表格「标题」列 | 直接使用 |
| 所属目录 | 组合推导 | 见下方所属目录规则 |
| 标签 | 从章节/模块特征推导 | 见下方标签推导规则 |
| 优先级 | Markdown 表格「优先级」列 | 直接取（P0/P1/P2/P3） |
| 用例类型 | 从章节类型推导 | 见下方用例类型规则 |
| 前置条件 | Markdown 表格「前置条件」列 | `<br>` 替换为 `\n` |
| 步骤 | Markdown 表格「测试步骤」列 | `<br>` 替换为 `\n` |
| 预期结果 | Markdown 表格「预期结果」列 | `<br>` 替换为 `\n` |
| 功能模块 | Markdown frontmatter `module` 字段 | 取 module 值 |

### 3. 所属目录规则

格式：`项目名|平台分类|模块名[|子模块|用例划分模块]`

**基础层级**（始终存在）：
- **项目名**：取自 frontmatter 的 `program` 字段（如 `MINISPHOTO`）
- **平台分类**：取自 frontmatter 的 `platform` 字段推导
  - `iOS App` / `Android App` / 包含 `App` → `app`
  - `Web` → `web`
  - `桌面端` / `Desktop` → `pc`
  - 全平台 / 未指定 → `all`
- **模块名**：取自 frontmatter 的 `module` 字段（如 `相册备份`）

**扩展层级**（仅当存在子章节时追加）：
- 如果 MD 文件有 `## 父章节` + `### 子章节` 的嵌套结构，所属目录扩展为：`项目名|平台分类|模块名|父章节|子章节`
  - 例：`## 十、非功能性测试` + `### 性能测试` → `...|非功能性测试|性能测试`
  - 例：`## 移动端专项用例` + `### 手势操作` → `...|移动端专项|手势操作`
- 如果只有单层 `##` 章节，所属目录为：`项目名|平台分类|模块名`（不追加）

### 4. 标签推导规则

根据当前用例所在的章节名称或模块名称中的关键词匹配：

| 关键词 | 标签 |
|--------|------|
| 权限 / 登录 / 角色 / 授权 | `权限` |
| 设置 / 配置 / 目的地整理 / 网络条件 | `配置` |
| 备份 / 执行 / 触发 / 进度 / 手动备份 / 自动备份 | `备份` |
| 异常 / 中断 / 恢复 / 错误 / 网络断开 / 空间不足 / 杀死进程 / 可靠 / 断点续传 | `异常` |
| 性能 / 速度 / 刷新 / 上传速度 | `性能` |
| 安全 / 加密 / EXIF | `安全` |
| 兼容 / 格式 / iOS / Android / JPEG / HEIC / RAW | `兼容性` |
| 手势 / 滑动 / 长按 | `手势` |
| 屏幕 / 刘海屏 / 横竖屏 / 小屏 / 大屏 / 布局 | `屏幕适配` |
| 系统交互 / 来电 / 深色模式 / 低电量 / 权限弹窗 | `系统交互` |
| 网络 / WiFi / 蜂窝 / 飞行模式 / 弱网 | `网络` |
| 生命周期 / 冷启动 / 热启动 / 后台 / 切后台 | `生命周期` |
| 多设备 / 同一目录 | `多设备` |
| 持久化 / 重启 / 缓存清理 | `持久化` |
| 状态流转 / idle / running / paused / completed | `状态流转` |
| 格式混合 / 多格式 | `格式` |
| 无障碍 / VoiceOver / 字体放大 | `无障碍` |
| 输入方式 / 软键盘 / 剪贴板 | `输入方式` |
| 存储 / 缓存 | `存储` |

### 5. 用例类型规则

根据用例所在章节类型推导：

| 章节特征 | 用例类型 |
|----------|----------|
| `## 移动端专项用例` 下的子章节 | `移动端用例` |
| `## 兼容性测试` 或包含兼容性关键词的章节 | `兼容性用例` |
| `### 性能测试` 子章节 | `性能用例` |
| `### 安全性测试` 子章节 | `安全用例` |
| 其他功能相关章节（权限/设置/执行/异常等） | `功能用例` |

一个用例可能同时属于多个类型（如既在兼容性章节又在移动端章节），用 `|` 分隔。

### 6. 格式处理

- `<br>` → `\n`：将 Markdown 表格中的 HTML 换行符替换为实际换行符

### 7. 输出到飞书表格

使用 `lark-sheets` 技能将数据写入飞书电子表格：

#### 7.1 创建新表格

使用 `lark-cli sheets +create` 创建新电子表格：

```bash
lark-cli sheets +create \
  --title "{原文件名}_测试用例" \
  --values '[["用例名称","所属目录","标签","优先级","用例类型","前置条件","步骤","预期结果","功能模块"], {数据行...}]'
```

- 第一行为表头（固定列名）
- 后续每行对应一条测试用例数据
- 如果用户指定了已有的飞书表格 URL/token，则使用 `+write` 或 `+append` 写入该表格

#### 7.2 写入已有表格

如果用户提供了已有表格 URL 或 token：

```bash
# 追加模式
lark-cli sheets +append --url "<表格URL>" --values '[[数据行1], [数据行2], ...]'

# 或覆盖写入
lark-cli sheets +write --url "<表格URL>" --sheet-id "<sheet_id>" --range "A2" --values '[[数据行1], ...]'
```

#### 7.3 设置表头样式（可选）

```bash
lark-cli sheets +set-style --url "<表格URL>" --range "A1:I1" --style '{"fontWeight":"bold","backgroundColor":"#f0f0f0"}'
```

#### 7.4 输出结果

操作完成后，向用户返回飞书表格的 URL，方便用户直接访问查看。

## 注意事项

- **必须使用飞书 skill 操作表格**：数据写入必须通过 `lark-sheets` skill 完成，禁止使用 Python/Node 等脚本直接调用 API
- 前置条件中多条用 `\n` 分隔，保持编号格式（如 `1. xxx\n2. xxx`）
- 步骤和预期结果必须一一对应，步数 = 预期结果数
- 推导标签和用例类型时，匹配所有关键词而非仅第一个，确保不漏标签
- 飞书表格的单元格内容不支持 BOM 编码，换行符直接使用 `\n`
- 如果表格数据量较大（超过 1000 行），建议分批写入，每次不超过 500 行
